Never use `pytest` - use `unittest` for running python tests. Use project root for testing.

If any changes are made to CLI interface or output format, run `./test_all.sh` to fix broken tests.

`plan.py` is is a generated file - do not edit!
Whenever any modifications to `plan.py` are needed, edit files in `src/` and run `./build.sh` afterwards.

Updates to documentation should touch `*.md` files in the project root, `src/help` and `src/claude`.
